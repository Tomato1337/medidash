# Search Service

## Назначение

Search Service обеспечивает поиск по медицинским документам:

- **Семантический поиск** через pgvector (поиск по смыслу)
- **Лексический поиск** (полнотекстовый поиск по ключевым словам)
- **Гибридный поиск** (комбинация семантического + лексического)
- **Фильтрация** по тегам, датам, типам документов
- **Ранжирование результатов**
- **Кэширование** популярных запросов в Redis

## Технологии

- NestJS
- Prisma ORM
- pgvector (vector similarity search)
- PostgreSQL Full-Text Search
- Redis (кэширование)
- OpenAI API (для векторизации поисковых запросов)

## Типы поиска

### 1. Семантический поиск
```typescript
// Поиск по смыслу, даже если слова не совпадают
query: "высокое давление"
// Найдет: "гипертония", "артериальная гипертензия"
```

Процесс:
1. Векторизация поискового запроса
2. Поиск похожих векторов в DocumentChunk
3. Ранжирование по cosine similarity

### 2. Лексический поиск
```typescript
// Традиционный поиск по ключевым словам
query: "анализ крови"
// Найдет документы со словами "анализ" и "крови"
```

### 3. Гибридный поиск
Комбинирует результаты обоих методов с весовыми коэффициентами.

## Порт

По умолчанию: **3004**

## Переменные окружения

```env
SEARCH_SERVICE_PORT=3004
DATABASE_URL=postgresql://...
REDIS_HOST=localhost
REDIS_PORT=6379
OPENAI_API_KEY=sk-...
```

## API Endpoints

- `GET /search/semantic` - Семантический поиск
- `GET /search/lexical` - Лексический поиск
- `GET /search/hybrid` - Гибридный поиск
- `GET /search/by-tags` - Поиск по тегам
- `GET /search/by-date-range` - Поиск по диапазону дат

## Параметры поиска

```typescript
{
  query: string;           // Поисковый запрос
  userId: string;          // ID пользователя
  limit?: number;          // Макс. кол-во результатов (default: 10)
  offset?: number;         // Пропустить результаты (pagination)
  tags?: string[];         // Фильтр по тегам
  dateFrom?: Date;         // Дата от
  dateTo?: Date;          // Дата до
  semanticWeight?: number; // Вес семантики (0-1)
  lexicalWeight?: number;  // Вес лексики (0-1)
}
```

## Кэширование

Популярные запросы кэшируются в Redis на 1 час.
Инвалидация кэша при добавлении новых документов.
